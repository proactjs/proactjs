<static> currentCaller :Object

Represents the current caller of a method, the initiator of the current action.

This property does the magic when for example an ProAct.AutoProperty is called for the first time and the dependencies to the other properties are created. The current caller expects to be used in a single threaded environment.

Do not remove or modify this property manually.

<static> flow :ProAct.Flow

The ProAct.Flow instance used by ProAct's property updates by default.

It defines only one queue - the default one proq.

It has default error callback that outputs errors to the ProAct.flow.errStream, if defined.

Override this instance if you are creating a framework or toolset over ProAct.js.

<static> prob(object, meta) → {Object}

The ProAct.prob method is the entry point for creating reactive values in ProAct.js

If the value is Number/String/Boolean/null/undefined or Function a new ProAct.Val is created woth value, set to the passed object value. The meta-data passed is used in the creation process.

If the passed object is an array, the result of this method is a new ProAct.Array with content, the passed array object

If the object passed is a plain JavaScript object the result of this function is reactive version of the object with ProAct.ObjectCore holding its ProAct.Propertys.

Parameters:
Name Type Description
object Object The object/value to make reactive.
meta Object | String Meta-data used to help in the reactive object creation.
