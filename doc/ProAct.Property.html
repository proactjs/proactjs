Class: Property

ProAct. Property

new Property(proObject, property, getter, setter)

Constructs a ProAct.Property. The properties are simple ProAct.Observables with state. The basic property has a state of a simple value - number/string/boolean.

Every property represents a field in a plain javascript object. It makes it reactive, on reading the property value, if ProAct.currentCaller is set, it is added as a listener to the property changes.

Every property has a type the default property has a type of a simple value.

All the properties of an object are managed by its ProAct.ObjectCore, which is set to a hidden field of the object - '__pro__'.

When created every property is in ProAct.States.init state, when it is functional, the state is changed to ProAct.States.ready. If the property is not in ProAct.States.ready state, it is not useable.

ProAct.Property#init is called by this constructor for the property initialization. It should initialize the property and set its state to ProAct.States.ready.

ProAct.Property is part of the properties module of ProAct.js.

Parameters:
Name Type Description
proObject Object A plain JavaScript object, holding a field, this property will represent.
property String The name of the field of the object, this property should represent.
getter function An optional getter to be used when the property is read. If this parameter is empty, a new ProAct.Property.defaultGetter is build for this.
setter function An optional setter to be used when the property is written. If this parameter is empty, a new ProAct.Property.defaultSetter is build for this.

<static> reProb(property) → {ProAct.Property}

Recreates a property, using its current value.

For example if the initial value of the field was null, the property can be set to be instance of ProAct.NullProperty, but if it changes to the number 3 it can be changed to ProAct.Property using this method.

The re-definition works by using ProAct.Property#destroy to destroy the passed property first, and then the ProAct.ObjectCore#makeProp method is called of the ProAct.ObjectCore of the object the property belongs to.

This way a new ProAct.Property instance is created to replace the passed one.

Links source observables into this observable. This means that this observable is listening for changes from the sources.

willUpdate(source, actions, eventData) → {ProAct.Observable}

Will update is the method used to notify observers that this ProAct.Observable will be updated.

It uses the ProAct.Observable#defer to defer the listeners of the listening ProAct.Observables. The idea is that everything should be executed in a running ProAct.Flow, so there will be no repetative updates.

The update value will come from the ProAct.Observable#makeEvent method and the source parameter will be passed to it.

If this ProAct.Observable has a parent ProAct.Observable it will be notified in the running flow as well.

Parameters:
Name Type Description
source Object The source of the update, for example update of ProAct.Observable, that this is observing.

Can be null - no source.

In the most cases ProAct.Event is the source.

actions Array | String A list of actions or a single action to update the listeners that listen to it. If there is no action provided, the actions from ProAct.Observable#defaultActions are used.
eventData Array Data to be passed to the event to be created.
